# VirtualizedList 虚拟化列表

> 待撰写：虚拟化列表详细设计

## 职责

提供虚拟化渲染的列表组件，只渲染可见区域。

## 什么是虚拟化？

传统列表渲染所有项目，当列表很长时（如 1000+ 条消息），会导致：
- 内存占用高（所有 React 组件都在内存中）
- 渲染性能差（初始渲染和更新都慢）

虚拟化列表只渲染可见区域的项目：
- 根据滚动位置计算可见项
- 只创建可见项的 React 组件
- 滚动时动态替换渲染内容

## 核心概念

- **Viewport**: 可见区域
- **Overscan**: 缓冲区（预渲染上下几项，避免滚动闪烁）
- **Item Height**: 每项高度（可固定或动态计算）
