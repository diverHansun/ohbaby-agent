# TextPart 文本内容组件

## 一、职责

TextPart 渲染 AI 响应中的文本内容，支持 Markdown 格式。它是出现频率最高的 Part 类型，负责将 Markdown 文本转换为终端可视化输出。

## 二、Markdown 渲染方案

采用 **Markdown parser + 自定义 Ink 渲染** 方案：

1. 使用 `remark`（或类似 Markdown parser）将 Markdown 文本解析为 AST
2. 遍历 AST 节点，用自定义 Ink 组件逐节点渲染

### 2.1 支持的 Markdown 元素

| 元素 | 渲染方式 |
|------|---------|
| 段落 | 普通文本，段落间空行 |
| 标题 | 加粗 + 颜色区分 |
| 代码块 | 语法高亮（基于语言标识），缩进显示，背景色区分 |
| 行内代码 | 反色或加亮显示 |
| 加粗 | 终端 bold 属性 |
| 斜体 | 终端 italic 属性（部分终端不支持，降级为普通文本） |
| 列表 | 有序/无序，缩进层级 |
| 链接 | 显示文本 + 下划线，URL 用弱化颜色附在后面 |
| 引用块 | 左侧竖线标记 + 缩进 |

### 2.2 MVP 不支持的元素

- 表格（终端对齐复杂，后续可扩展）
- 图片（终端无法显示）
- 水平分隔线（可后续添加）

### 2.3 代码块语法高亮

代码块根据语言标识（如 ```typescript）使用语法高亮库（如 `highlight.js` 的终端适配）进行着色。不识别的语言降级为纯文本显示。

## 三、流式更新

TextPart 支持流式文本渲染：

1. TextPart 组件挂载时，订阅 `Message.Event.PartUpdated` Bus 事件
2. 事件 payload 中 `partId` 匹配当前 Part 的 `id` 时处理
3. 从 payload 的 `delta` 字段获取增量文本
4. 将增量文本追加到已有文本，重新解析 Markdown 并渲染

### 3.1 性能考量

每次增量到达都重新解析全量 Markdown 并不理想，但 MVP 阶段文本量可控，remark 解析速度足够快。如果未来出现性能瓶颈，可以考虑增量解析优化。

## 四、数据输入

| 字段 | 类型 | 来源 | 说明 |
|------|------|------|------|
| id | string | Part 基础字段 | 用于匹配 Bus 事件 |
| text | string | TextPart 字段 | Markdown 文本内容 |
| synthetic | boolean（可选） | TextPart 字段 | 系统生成的文本，可能需要不同样式 |

## 五、设计约束

1. **不负责布局**：TextPart 只渲染文本内容，宽度由父组件容器决定
2. **不处理用户交互**：文本内容不可点击、不可选中（终端限制）
3. **不缓存 AST**：MVP 阶段每次更新重新解析，简化实现
4. **自行订阅 Bus**：流式更新通过直接订阅 Bus 事件实现，不依赖父组件 props 传递增量
