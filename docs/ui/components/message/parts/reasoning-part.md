# ReasoningPart 推理内容组件

## 一、职责

ReasoningPart 渲染 AI 的推理/思考过程。推理内容通常较长且用户不一定需要查看，因此默认折叠显示，可展开查看完整内容。

## 二、视觉结构

### 折叠态（默认）

```
> Thinking... (click to expand)
```

显示一行折叠提示，使用弱化颜色。可以附带推理文本的前几个字作为摘要预览。

### 展开态

```
v Thinking:
  用户要求重构 auth 模块，我需要先看一下现有的代码
  结构，然后分析主要问题...（完整推理文本）
```

展开后显示完整的推理文本，使用弱化颜色与正式响应文本区分。

## 三、交互

- 折叠/展开通过键盘或鼠标触发
- 展开状态存储在组件内部 state（`useState`），不写入 Context
- 每个 ReasoningPart 独立管理自己的折叠状态

### 3.1 基于 shared/Collapsible

ReasoningPart 使用 `shared/Collapsible` 组件包裹推理内容，Collapsible 提供折叠/展开的通用能力。

## 四、数据输入

| 字段 | 类型 | 来源 | 说明 |
|------|------|------|------|
| id | string | Part 基础字段 | Part 唯一标识 |
| text | string | ReasoningPart 字段 | 推理文本内容 |
| time | `{ start, end }` | ReasoningPart 字段 | 推理的时间范围（可选展示） |

## 五、设计约束

1. **默认折叠**：减少屏幕占用，用户按需展开
2. **纯文本渲染**：推理内容不解析 Markdown，直接显示原始文本（推理内容通常没有格式化需求）
3. **弱化显示**：折叠和展开态都使用弱化颜色（dimColor），与正式响应文本形成视觉层次
4. **不参与流式更新**：推理内容通常在响应开始时完整接收，不需要流式增量渲染
