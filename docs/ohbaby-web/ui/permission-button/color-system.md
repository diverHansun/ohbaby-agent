# Permission Buttons · 配色系统

> 权限弹窗 4 个按钮的配色规格。原则:**复用现有低饱和 token,不凭空造色**。整站正文 `IBM Plex Sans`,背景 `#fafafa`,正文色 `#26282d`,既有唯一"危险"色是 stop 按钮的柔和砖红 `#c4756b`——新红色贴着它,不用刺眼的纯红。

---

## 1. 后果梯度

颜色按"后果轻重"编码,三档 + 一个安静的次级允许:

| 后果档 | 含义 | 视觉 |
| --- | --- | --- |
| 主/默认 | 批准这一次 | 实心蓝(唯一高对比) |
| 次/安静的允许 | 批准并记住 | 浅蓝描边 |
| 警示 | 良性拒绝(run 继续) | 淡琥珀 |
| 危险 | 终止整个 run | 描边红 + 半透明红填充 |

## 2. Token 表

| 按钮 | 角色 | 背景 | 描边 | 文字 | hover 背景 | 来源 |
| --- | --- | --- | --- | --- | --- | --- |
| Allow once | 主 | `#5f86c4` | `#5278bb` | `#ffffff` | `#5278bb` | 现有 primary,不变 |
| Always allow | 次 | `#eef2f9` | `#d8e1f0` | `#4a6ba6` | `#e3ebf6` | 复用 `ohb-mode-plan` 蓝 |
| Reject | 警示 | `#faf3e4` | `#ead7ad` | `#8f6f2f` | `#f4ead2` | 新增柔和琥珀,贴合低饱和语言 |
| Cancel run | 危险 | `rgba(196,117,107,0.16)` | `#c4756b` | `#a0493d` | `rgba(196,117,107,0.24)` | 贴 stop 按钮 terracotta `#c4756b` |

## 3. 危险色(Cancel run)细则

- 描边用**实线** terracotta `#c4756b`,清晰可辨。
- 内部填充用**半透明红**而非实心色块,以表达"危险但不尖叫"。
- 起始 alpha = `0.16`,**这是实现时唯一需要现场微调的旋钮**:觉得太淡就往 `0.20`–`0.24` 提;不要改成完全实心红(与整体描边语言冲突)。
- 文字用更深的 terracotta `#a0493d`,保证在淡红填充上的对比度。

## 4. 约束

- 这 4 组色值只用于权限弹窗专用类(见 [implementation.md](./implementation.md)),不污染 `.ohb-button` / `.ohb-button-primary`。
- 不抽公共 CSS 变量;沿用站内现有的内联 hex/rgba 习惯。
- 4 个按钮 = 1 实心 + 3 浅色,整体仍是"一个锚点 + 三个安静档"的克制版式,不做成彩虹。

## 5. Acceptance

- 4 个按钮呈现表中 4 种处理,同屏仅 1 个实心主按钮。
- `Reject`(琥珀)与 `Cancel run`(红)颜色明显不同,肉眼可区分后果。
- 危险色为半透明填充 + 实描边,非纯实心红。
- `styles.unit.test.ts` 对 4 个类的 `background` / `border-color` / `color` 有断言。
