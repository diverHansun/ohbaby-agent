# system-prompt improve-1 设计方案

> 日期：2026-05-29
> 分支：`codex/system-prompt-improve-1`
> 范围：`packages/ohbaby-agent/src/core/system-prompt/`

---

## 1. 目标

本轮目标是把系统提示词模块的代码事实、职责边界和文档对齐，并把高频编辑的静态提示词从 TypeScript 长字符串迁移到 `.md` 模板文件。

本轮不是 prompt 系统重写，不引入 provider/modelFamily overlay，不引入 Nunjucks/profile renderer，不做 runtime hot reload。

---

## 2. 模块职责

`system-prompt` 负责：

- 统一存储 primary/subagent 静态 prompt 模板。
- 按 primary/subagent 边界组装有序 prompt layer。
- 渲染 agent addon、tool guidance、environment、custom instructions。
- 过滤可疑 tool snippets，并安全加载 custom instructions。
- 通过 `createSystemPromptProvider()` 适配 context manager。

`system-prompt` 不负责：

- agent profile、tools、permission、maxSteps 等代理配置。
- LLM provider/model 选择。
- 工具注册、权限审批、并发调度或执行。
- 动态生成 prompt。
- memory 注入与消息历史管理。

---

## 3. 设计决策

### 3.1 `.md` 作为模板载体

所有静态 prompt 内容迁移到 `prompts/` 下的 `.md` 文件。`.md` 可以包含 Markdown、XML block 或两者混合。

TS wrapper 只负责：

1. 从 `templates.generated.ts` 读取生成后的模板常量。
2. 导出稳定常量或 lookup 函数。
3. 由测试保证生成快照与 `.md` 内容一致。

### 3.2 构建时内联

模板通过 `scripts/generate-system-prompt-assets.mjs` 生成 checked-in TS 快照。运行时不新增 `fs` 读取静态模板，避免发布包路径问题，也兼容 `tsx` 直接运行源码的开发和集成测试路径。

### 3.3 保持分层组装

primary 顺序：

```text
identity -> task -> agent addon -> tools -> environment -> custom
```

subagent 顺序：

```text
subagent base -> subagent task -> agent addon -> tools -> minimal environment
```

继续使用 assembler 中的数组字面量表达顺序，不新增 `LAYER_ORDER + sort`。

### 3.4 删除无语义 API

`generateAgentPrompt()` 是恒等函数，删除该文件和公共导出。`assembler.ts` 直接把 trimmed addon 包进 `<agent_prompt_addon>`。

`AssembleResult` 没有消费者，且 `assemble()` 并不返回该类型，删除类型和公共导出。

## 4. 实施波次

### Wave 1: 低风险清理

- 删除 `layers/agent.ts`。
- 删除 `generateAgentPrompt` 公共导出。
- 删除 `AssembleResult` 类型。
- 增加公共 API 回归测试。

### Wave 2: 模板外置

- 新增 `.md` 模板资产。
- 新增生成脚本。
- 新增 `prompts/templates.generated.ts`。
- 新增 `prompt:generate` / `prompt:check`，并让 build 先检查快照同步。
- TS prompt wrapper 改为读取生成快照。
- 新增模板存在、非空、导出等价测试。

### Wave 3: 文档对齐

- 更新 `goals-duty.md`、`architecture.md`、`data-model.md`、`dfd-interface.md`、`test.md`。
- 明确 tools 是条件层。
- 明确 `assemble()` 返回 `string[]`。

### Wave 4: 暂缓方向

- provider/modelFamily overlay。
- kimi-style profile renderer。
- Nunjucks 或其他模板引擎。
- runtime hot reload。
- custom loader 物理归位到 services。

---

## 5. 验收标准

- `pnpm exec vitest run packages/ohbaby-agent/src/core/system-prompt` 通过。
- `pnpm run typecheck` 通过。
- `pnpm --filter ohbaby-agent build` 通过。
- `.md` 模板文件存在、非空，且与导出内容一致。
- primary/subagent 层顺序有直接数组断言。
- 子代理仍不加载 primary custom instructions。
- 文档不再描述旧四层结构或不存在的结果模型。
