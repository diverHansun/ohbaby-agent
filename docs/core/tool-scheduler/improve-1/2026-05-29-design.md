# tool-scheduler improve-1 design

Date: 2026-05-29
Branch: `codex/tool-scheduler-improve-1`
Scope: `packages/ohbaby-agent/src/core/tool-scheduler`, skill tools, MCP adapters, and UI runtime helper extraction.

## Goals

- Keep `ToolSource` as origin metadata, not behavior metadata.
- Give skill tools their own source: `source: "skill"`.
- Replace scheduler-owned MCP trust checks with a generic tool field: `requireExplicitApproval`.
- Keep MCP `isTrusted` as MCP-local metadata only.
- Extract UI runtime stream bridge conversion and unknown-value formatting from `composition.ts` without splitting the composition root.

## Target Model

```ts
type ToolSource = "builtin" | "module" | "skill" | "mcp";

interface Tool {
  source: ToolSource;
  category?: ToolCategory;
  requireExplicitApproval?: boolean;
}
```

`ToolSource` answers where a tool comes from. `ToolCategory` answers how scheduler policy and concurrency should treat it.

`requireExplicitApproval: true` means the scheduler must call `Permission.ask()` even when the current permission state already allows the category. The ask uses `reason: "explicit-approval-required"` and `rememberable: false`.

## Implementation Waves

### Wave 1: source/category correction

- Add `"skill"` to `ToolSource`.
- Make `skill` and `skill_resource` register with `source: "skill"`.
- Infer `category: "skill"` for `source: "skill"` when no explicit category is provided.

### Wave 2: composition helper extraction

- Add `runtime-format.ts` for `formatUnknown`.
- Add `stream-bridge-run-event-source.ts` for stream bridge event conversion.
- Keep service construction, refresh logic, and wiring in `composition.ts`.

### Wave 3: generic explicit approval

- Add `requireExplicitApproval?: boolean` to `Tool`.
- Map MCP `trust: false` to `requireExplicitApproval: true`.
- Set `mcp_resource` and `mcp_prompt` to `requireExplicitApproval: true`.
- Remove scheduler checks for `source === "mcp"` and `isTrusted !== true`.

## Deferred

- Moving `ToolExecutionEnvironment` out of `tool-scheduler`.
- Resource-aware scheduler.
- Model-aware tool projection.
- A larger composition root redesign.
