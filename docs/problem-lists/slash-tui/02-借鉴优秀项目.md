# 2. 借鉴优秀项目

## 2.1 opencode

### 架构模式

opencode 的 TUI slash-command 是**纯前端驱动**的：

```
命令注册 → Dialog Modal → 组件直接读 reactive store → JSX 渲染
```

没有 "命令 emit 数据 → TUI 格式化展示" 的中间契约层。每个命令是独立的 React 组件（ISX），打开一个 modal overlay，通过 `useSync().data` 直接获取所需数据。

### 值得借鉴的设计

#### (1) `/mcps` 交互式管理

`DialogMcp` 使用 `DialogSelect` 渲染 MCP 服务器列表，每行显示：
- 服务器名称（`title`）
- 连接状态描述（`description`，如 "connected"、"failed"）
- 右侧 Enabled/Disabled 状态指示器（`✓ Enabled` 绿色加粗 / `○ Disabled` 灰色）
- 支持 Space 键切换启用/禁用（调用 `local.mcp.toggle()` 后刷新状态）

opencode 的 MCP 状态值有 5 种：`connected`、`failed`、`disabled`、`needs_auth`、`needs_client_registration`。

#### (2) `/status` 分区域展示

`DialogStatus` 不使用 `DialogSelect`，而是用自定义 JSX 分区渲染：
- MCP Servers section（count + 每行 name + 彩色状态点 + 详情）
- LSP Servers section
- Formatters section
- Plugins section

状态点颜色映射：`connected→success(绿)`、`failed→error(红)`、`disabled→textMuted(灰)`、`needs_auth→warning(黄)`、`needs_client_registration→error(红)`。

#### (3) Autocomplete 双源合并

opencode 的 autocomplete 合并两个来源：
1. TUI 内置 command（通过 `command.register()` 注册）
2. 服务端 command（通过 `sync.data.command` 获取）

#### (4) `/help` 是极简提示

`DialogHelp` 仅显示一行提示 "Press [keybind] to see all commands" + OK 按钮。**不列出命令**——实际的命令列表通过 `command_list` 快捷键（Ctrl+P）打开 `DialogCommand` 展示。

#### (5) `/skills` 列表选择

`DialogSkill` 使用 `DialogSelect` + `dialog.setSize("large")`，通过 SDK 获取技能列表，每行显示 name（对齐填充）+ description，选中后将 `/skillname` 文本插入 prompt。

### 不适用于 ohbaby 的模式

opencode 的 "命令 = React 组件" 模式**不适用**，因为 ohbaby-agent 是事件驱动架构，命令执行在 agent 端，TUI 只是消费者。保留 `dataOutput(subject, data)` 契约模式是正确的。

## 2.2 kimi-code

### 架构模式

kimi-code 的 slash-command 是**终端框架直接渲染**的（非 React/Ink）：

```
命令 handler → 构建预着色 string[] → UsagePanelComponent(行数组) → transcriptContainer.addChild()
```

核心渲染元件：
- `UsagePanelComponent(lines, borderColor, title?)` — 带边框的盒子，包裹预着色文本行
- `NoticeMessageComponent(title, detail)` — spacer + 标题（加粗）+ 详情（dim）
- `StatusMessageComponent(message, color?)` — 单行着色文本

### 值得借鉴的设计

#### (1) `/status` 输出的结构化行构建

`buildStatusReportLines()` 按顺序构建行数组（status-panel.ts:91-148）：

```
>_ Kimi Code (v1.2.3)

  Model         gpt-5.5 (thinking on)
  Directory     /home/user/project
  Permissions   auto
  Plan mode     off
  Session       session_1

Context window
  ████████░░  73.0%  (73k / 100k)
```

特点：
- 标题行使用 `>_ ProductName (vX.Y.Z)` 格式
- 字段行使用对齐填充（`label.padEnd(labelWidth)`）
- Context window 独立为一个 section，带进度条
- 可选 `Title` 和 `Warning` 行
- 末尾可追加 `ManagedUsageReport`（计划配额）

#### (2) `/mcp` 表格化展示

`buildMcpStatusReportLines()` 生成表格行（mcp-status-panel.ts:85-143）：
```
Servers
  Name        Status       Transport   Tools
  github      connected    streamable   5 tools
  filesystem  failed       streamable   3 tools
    error: Connection refused
  postgres    needs auth   streamable   2 tools
    action: run /mcp-config login postgres

  2 connected · 1 failed · 8 tools available
```

特点：
- 表头行使用 muted 颜色
- 按状态优先级排序（failed 排最前）
- failed 服务器有缩进的 `error:` 子行
- needs-auth 服务器有 `action:` 子行提示修复命令
- 末尾有汇总行（`N connected · N failed · N tools available`）

#### (3) `/usage` 命令（kimi-code 独有）

`buildUsageReportLines()` 展示 session token 用量：
- 按模型分组的 input/output/total token 统计
- Context window 进度条
- 可选的 ManagedUsageReport（计划配额进度条）

### 不适用于 ohbaby 的模式

kimi-code 使用**预着色 chalk 字符串数组**，ohbaby-agent 应使用**结构化数据**（`dataOutput`），让 TUI 端决定如何着色。这是 ohbaby 事件驱动架构的优势——数据与展示分离。

### kimi-code 的 scroll windowing

`HelpPanelComponent` 实现 viewport-based scrolling（`scrollTop`, `maxVisible`），带 "showing X-Y of Z" 指示器。相关：当命令列表过长时支持 PgUp/PgDn 分页。

### kimi-code 的 command priority + availability

命令有 `priority` 数值排序（100, 80, 60, 40, 20），以及 `availability` 控制（`'always'`、`'idle-only'`、或函数 `(args) => availability`）。

## 2.3 核心借鉴总结

| 借鉴项 | 来源 | 应用方式 |
|--------|------|----------|
| 数据按段/分类组织 | opencode `/status` | `/status` data 按 `model`, `session`, `tools`, `context` 段组织 |
| 列表项含状态字段 | opencode `/mcps` | `/mcps` data 每项含 `status` 字段，为交互预留 |
| 结构化数据而非预格式化字符串 | kimi-code → 我们的推论 | 保留 `dataOutput("subject", structured)` 模式，TUI 端再映射为展示文本 |
| 分页/滚动能力预留 | kimi-code `/help` | `/help` 数据保持不变，TUI 端可选分页渲染（PgUp/PgDn） |
| 命令行 alias | opencode + kimi-code | 两个项目都支持 alias，`/mcps` 加 alias `/mcp`，`/skills` 加 alias `/skill` |
| 命令可见性元数据 | opencode `CommandOption` | `suggested`/`hidden`/`enabled` 字段可用于控制命令在 autocomplete 中的可见性 |
| 错误详情子行 | kimi-code `/mcp` | failed 项可附带 error 详情，needs-auth 项可附带 action 提示 |
| 进度条可视化 | kimi-code `/status` + `/usage` | context window 和 usage 配额使用 `renderProgressBar` |
