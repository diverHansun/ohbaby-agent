# 3. 实施计划

## 3.1 当前结论

本轮采用“方案 B”：保留现有 `ohbaby-sdk` slash-command 解析和 `ohbaby-agent` command service/provider 结构，只补齐后端命令输出字段和最小 TUI 文本格式化。

本轮不做以下事情：

- 不做 TUI 交互重设计、分页、弹窗、复杂布局。
- 不做 command domain DTO 全量标准化。
- 不新增 `/tools`。
- 不在 `/mcps` 中列具体 MCP tools、tool 数量或错误详情。
- 不在 `/skills` 中加入 `pluginId`，因为 plugin 模块尚未实施。

如果后续 command 输出字段继续膨胀，或者多个 surface 对同一命令输出出现重复转换，再借鉴 opencode、kimi-code 等项目评估是否需要统一 command domain DTO。

## 3.3 后端数据契约

### `/help`

保留原有 `commands`，新增 `categories`：

```typescript
dataOutput("help", {
  commands: UiCommandSpec[],
  categories: [
    {
      name: "system",
      title: "System",
      commands: UiCommandSpec[],
    },
  ],
});
```

说明：

- `commands` 保持向后兼容。
- `categories` 按 `UiCommandSpec.category` 分组。
- `name` 保留原始 category 值。
- `title` 只做简单可读化，例如 `system -> System`。
- 动态 skill slash command 也会被分入 `skill` category。

### `/status`

保留旧字段：

```typescript
{
  model: CommandModelSummary | null,
  models: CommandModelSummary[],
  status: string,
}
```

新增字段：

```typescript
{
  sessionId: string | null,
  tools: {
    builtin: number,
    module: number,
    skill: number,
    mcp: number,
  },
  skillsCount: number,
  mcps: {
    total: number,
    connected: number,
    failed: number,
    disabled: number,
    disconnected: number,
  },
  context: ContextUsage | null,
  projectRoot: string | null,
}
```

说明：

- `tools` 是工具来源计数，只统计精确 source：`builtin`、`module`、`skill`、`mcp`。
- `mcps` 是 MCP server 状态汇总，和 `tools.mcp` 不是同一个概念。
- `context` 直接使用 context 模块的 `ContextUsage` 字段，例如 `currentTokens`、`contextLimit`、`remainingTokens`、`usageRatio`。
- provider 缺失时使用空数组、0 计数或 `null`。
- MCP provider 抛错时，`/status` 降级为 MCP 空汇总，不让整条命令失败。

### `/mcps`

新增 `/mcps`，alias `/mcp`。只输出 MCP server 和状态：

```typescript
dataOutput("mcps", {
  servers: [
    { name: "github", status: "connected" },
    { name: "memory", status: "disabled" },
    { name: "bad", status: "failed" },
  ],
});
```

说明：

- 不输出具体 MCP tools。
- 不输出 `toolCount`。
- 不输出错误详情。
- 即使底层 provider 提供了额外字段，command 层也会白名单投影成 `{ name, status }`。
- provider 缺失或抛错时返回 `{ servers: [] }`。

### `/skills`

新增 `/skills`，输出用户可调用 skill，并与动态 skill slash command 对齐：

```typescript
dataOutput("skills", {
  skills: [
    {
      name: "brainstorming",
      description: "Brainstorm ideas into designs and specs",
      path: ["brainstorming"],
      commandId: "skill.brainstorming",
      scope: "user",
      source: "codex-home",
    },
  ],
});
```

说明：

- `scope` 必填，只能是 `user` 或 `project`。
- `source` 可选，来自 skill loader/provider，用来说明 skill 的发现来源。
- 不输出 `pluginId`。
- command 层会过滤非法 `scope`，避免坏 provider 把非法 skill 暴露给 `/skills` 或 `/help`。

## 3.4 类型设计

核心新增类型：

```typescript
export type CommandSkillScope = "user" | "project";

export interface CommandSkillSummary {
  readonly name: string;
  readonly description: string;
  readonly scope: CommandSkillScope;
  readonly source?: string;
}

export type CommandMcpServerStatus =
  | "connected"
  | "failed"
  | "disconnected"
  | "disabled";

export interface CommandMcpServerSummary {
  readonly name: string;
  readonly status: CommandMcpServerStatus;
}

export interface CommandMcpProvider {
  listServers():
    | Promise<readonly CommandMcpServerSummary[]>
    | readonly CommandMcpServerSummary[];
}
```

`CommandServiceOptions` 新增：

```typescript
readonly mcps?: CommandMcpProvider;
readonly getContextUsage?: (input: {
  readonly sessionId?: string;
}) => Promise<ContextUsage | null> | ContextUsage | null;
readonly getProjectRoot?: () => Promise<string> | string;
```

## 3.5 适配器接线

`ui-runtime/composition.ts` 提供：

```typescript
getContextUsage(input: {
  readonly projectRoot: string;
  readonly sessionId: string;
}): Promise<ContextUsage>;

listMcpServerSummaries(): Promise<readonly CommandMcpServerSummary[]>;
```

`ui-inprocess.ts` 接入到 command service：

- `tools.listTools()` 继续使用 `runtime.listToolSummaries()`。
- `skills.listUserInvocable()` 使用已有 skill registry 结果，保留 `scope` 和可选 `source`。
- `mcps.listServers()` 使用 `runtime.listMcpServerSummaries()`。
- `getProjectRoot()` 使用已有 `resolveProjectRoot()`。
- `getContextUsage({ sessionId })` 只有在 sessionId 存在时组装 context usage；失败时返回 `null`。

## 3.6 TUI 最小格式化

只在 `packages/ohbaby-cli/src/tui/store/events.ts` 增加 command result 的可读文本：

- `help`: 优先显示 `categories`，否则回退 `commands` 或 JSON。
- `status`: 输出 status/model/session/tools/skills/mcps/context/project。
- `mcps`: 输出 `mcps: github connected, memory disabled`。
- `skills`: 输出 `skills: review [project, project-native] - Review code`。

这不是 TUI 设计优化，只是避免新后端结构在 UI 中直接显示成难读 JSON。

## 3.7 当前保留风险

- `COMMAND_NOTICE_TEXT_LIMIT` 仍会截断长输出，本轮不改通知窗口。
- `/status` 只做短 MCP server 汇总，不展示每个 server。
- `/mcps` 只列 server 状态，不提供 enable/disable 操作。
- plugin 模块未实施前，`/skills` 不表达 plugin 维度。
