# 4. 测试与验收标准

## 4.1 测试原则

- 优先写 unit tests 固化边界行为，再做真实 provider smoke。
- 时间相关测试必须使用 fake timers 或可注入 clock，不等待真实 5min/30min/1h。Vitest 场景优先使用 `vi.useFakeTimers()` + `vi.advanceTimersByTime()`。
- 网络 retry 测试使用 mock provider，不真实打满 5 次 API。
- 真实 `.env` API key 只用于 smoke，不打印 key，不把 key 写入 snapshot。
- 验收重点是“run 不过早终止”，不是“每个工具永不失败”。

## 4.3 关键测试用例

### maxSteps 文本收束

验收条件：

- 当 `maxSteps = 2` 时，第 2 次 model call 的 tools 为空数组。
- 结果为 succeeded 或明确的 `max_steps_finalized`，不是普通 failed。
- final response 包含已完成工作、剩余工作、下一步建议。

命令：

```powershell
pnpm exec vitest run packages/ohbaby-agent/src/core/lifecycle/lifecycle.unit.test.ts -t "maxSteps|final" --passWithNoTests
```

### 同步子代理 5min timeout

验收条件：

- `AgentService` 是同步子代理 5min deadline 的权威来源。
- scheduler `task` guard timeout 解析为 `310_000ms`，比 service deadline 多 10s cleanup grace。
- `AgentConfig.timeout = 600_000` 时同步子代理仍 clamp 到 `300_000ms`。
- 超时后 child run 被 cancel。
- 父 lifecycle 收到的是 tool result/error，不是父 run 自动 abort。
- tool metadata 保留 `sessionId`，允许用 `resume_session_id` 继续。

命令：

```powershell
pnpm exec vitest run packages/ohbaby-agent/src/agents/service.unit.test.ts packages/ohbaby-agent/src/tools/task.unit.test.ts -t "timeout|resume|subagent" --passWithNoTests
```

### 异步子代理 30min/60min timeout

验收条件：

- `agent_open` 很快返回 task id，不等待后台 run 完成。
- background task 默认 `timeoutMs = 1_800_000`。
- 运行时配置 `timeout = 3_600_000` 时允许 1h；schema 上限校验放到后续配置迁移。
- 超过 deadline 后状态为 `timed_out`，`agent_status` 能显示 timeout/error。
- 超时前已排队的 `agent_eval` 不会静默丢弃，会在 timeout 后启动下一轮。
- `timed_out` 后允许显式 `agent_eval` 继续同一个 child session。
- `agent_close` 对 `timed_out` 任务不会抛出非预期错误。

命令：

```powershell
pnpm exec vitest run packages/ohbaby-agent/src/agents/tasks/manager.unit.test.ts packages/ohbaby-agent/src/tools/agent-task.unit.test.ts -t "timeout|timed_out|background" --passWithNoTests
```

### provider retry

验收条件：

- 连接错误、408、429、500、502、503、504、529 会 retry；409 Conflict 不作为默认 retryable。
- abort、认证错误、参数错误不会 retry。
- 默认首次失败后最多 retry 5 次。
- retry delay 使用指数退避和 jitter。
- 有 `Retry-After` 时优先遵守，且不超过 `60_000ms`。
- retry 事件包含 attempt/maxRetries/delayMs/reason。
- mid-stream 已产生 delta 后失败，不自动重放，由 lifecycle 映射为 `provider_stream_interrupted` terminal reason。
- 用户 abort 不再伪装为 `finishReason: "length"`；本地原因写入 `streamStopReason: "user_aborted"`。

命令：

```powershell
pnpm exec vitest run packages/ohbaby-agent/src/core/llm-client/llm-client.test.ts packages/ohbaby-agent/src/core/lifecycle/lifecycle.unit.test.ts -t "retry|provider|stream" --passWithNoTests
```

## 4.4 集成测试

运行现有相关测试集：

```powershell
pnpm exec vitest run packages/ohbaby-agent/src/core/lifecycle/lifecycle.unit.test.ts packages/ohbaby-agent/src/core/agents/runner.unit.test.ts packages/ohbaby-agent/src/agents/service.unit.test.ts packages/ohbaby-agent/src/agents/tasks/manager.unit.test.ts packages/ohbaby-agent/src/core/tool-scheduler/scheduler.unit.test.ts packages/ohbaby-agent/src/tools/task.unit.test.ts packages/ohbaby-agent/src/tools/agent-task.unit.test.ts --passWithNoTests
```

期望：

- 所有测试通过。
- 旧的 maxSteps 硬失败断言已迁移到文本收束或结构化兜底。
- `task` 5min timeout 不影响普通工具 timeout。
- scheduler guard 不应先于 `AgentService` deadline 抢先返回，正常路径应产生结构化 subagent timeout。

运行 adapter contract：

```powershell
pnpm exec vitest run packages/ohbaby-agent/src/adapters/ui-inprocess.contract.test.ts -t "subagent|background agent task|parent prompt is aborted|maxSteps|timeout|retry" --passWithNoTests
```

期望：

- parent abort 仍能取消子代理。
- background task 不因 parent run 完成而取消。
- maxSteps/defaults 通过 adapter 贯通。

## 4.5 真实 `.env` smoke

仅在本地 `.env` 有真实 provider key 时运行。执行前确认命令不会打印 key。

建议新增或复用 smoke：

```powershell
pnpm run test:smoke:real
pnpm exec vitest run packages/ohbaby-agent/src/config/llm/__tests__/connect-model.real.e2e.test.ts --runInBand
```

真实验证场景：

- 正常短 prompt 可完成。
- provider 连接失败时显示 retry event。
- 人工配置错误 key 时不 retry 到长时间卡死，应快速失败为 non-retryable。
- 人工指向不可达 baseURL 时 retry 5 次后终止为 `provider_retry_exhausted`。
