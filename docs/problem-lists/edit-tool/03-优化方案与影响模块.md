在 `edit.ts` 和 `write.ts` 成功写入后，调用 `recordTextFileRead()` 更新全局读取状态，使 Map 中的 mtimeMs 与实际文件一致。

```typescript
// 现有代码（约第 119-120 行）
await writeTextFileAtomic(writePath, updatedContent);

// 新增一行

  context,
  mtimeMs: written.mtimeMs,
  resolvedPath: writePath,

```

需要新增 import：
```typescript

```

#### ② `packages/ohbaby-agent/src/tools/write.ts`

同理，在 `writeTextFileAtomic` 之后添加 `recordTextFileRead` 调用。write.ts 约在第 84-95 行。

### 不改动
- `read-state.ts`：不变
- `text-files.ts`：不变
- 测试：无需改动（现有测试验证的是"未读就编"拒绝行为，本次修复不影响该行为）

### 遗留问题
此方案只修复了当前 bug，但没有解决架构层面的问题：
- 双重检查仍然冗余
- 全局 Map 状态管理仍然存在
- mtimeMs 方案的固有问题（时钟精度、虚假阳性）未解决

## 3.3 方案B：去除冗余检查（建议中期推送）

### 原理
移除 `assertTextFileWasReadBeforeEdit` 检查，仅保留 `assertExpectedMtimeMs`。理由：
1. `assertExpectedMtimeMs` 已经要求 LLM 传递正确的 mtimeMs，足以证明 LLM 最近读取过文件
2. `old_string` 匹配失败本身就是最可靠的"内容已过时"信号
3. 消除全局 Map 的状态管理负担
4. kimi-code 和 opencode 都没有此机制

### 改动文件

#### ① `packages/ohbaby-agent/src/tools/edit.ts`

删除约第 67-72 行的 `assertTextFileWasReadBeforeEdit` 调用块：

```diff
- // After reading, verify the file was previously read in this session
- assertTextFileWasReadBeforeEdit({
-   actualMtimeMs: file.mtimeMs,
-   context,
-   inputPath,
-   resolvedPath: existingPath,
- });
```

同时移除文件顶部的 import：
```diff
- import { assertTextFileWasReadBeforeEdit } from './utils/read-state.js';
```

#### ② `packages/ohbaby-agent/src/tools/utils/read-state.ts`

移除 `assertTextFileWasReadBeforeEdit` 函数（~15 行）。保留 `recordTextFileRead` 函数（未来方案 C 可能用到）。

#### ③ `packages/ohbaby-agent/src/tools/edit.unit.test.ts`

移除 `"requires the file to be read in the same session before edit"` 相关测试用例（当前行 154-171 及 200-269）。改为验证：未提供 `expected_mtime_ms` 时被拒绝（由 check #1 负责）。

### 遗留问题
- mtimeMs 方案仍然存在（虚假阳性风险未消除）
- 无文件级并发锁（极端情况下两个 edit 可能交错）

## 3.4 方案C：架构重构（建议长期规划）

### 原理
用 opencode 的设计替换当前的 mtimeMs 方案：
1. 移除所有 mtimeMs 追踪逻辑和全局 Map
2. 添加文件级 Semaphore 锁防止并发编辑
3. 增加模糊匹配链提高容错性

### 改动文件

#### ① `packages/ohbaby-agent/src/tools/edit.ts`（核心改动）

新增文件级锁：
```typescript
import { Semaphore } from 'effect';

const fileLocks = new Map<string, Semaphore.Semaphore>();

function acquireFileLock(filePath: string): Semaphore.Semaphore {
  const resolved = resolve(filePath);
  const existing = fileLocks.get(resolved);
  if (existing) return existing;
  const sem = Semaphore.makeUnsafe(1);
  fileLocks.set(resolved, sem);
  return sem;
}
```

移除 mtimeMs 检查代码（约 10 行），移除 `expected_mtime_ms` 参数要求。

增强 `old_string` 匹配：添加行级 trim 匹配器、空白归一化匹配器（参考 opencode 的前 3-4 级）。

#### ② `packages/ohbaby-agent/src/tools/read.ts`

移除 `recordTextFileRead` 调用（约 4 行），移除 metadata 中的 `mtimeMs` 字段。

#### ③ `packages/ohbaby-agent/src/tools/write.ts`

添加文件级锁，移除 mtimeMs 相关代码。

#### ④ `packages/ohbaby-agent/src/tools/utils/read-state.ts`

整个文件可移除（48 行），或保留空壳标记为 deprecated。

#### ⑤ `packages/ohbaby-agent/src/tools/utils/text-files.ts`

移除 `assertExpectedMtimeMs` 函数（约 15 行），移除 `readWrittenFileMetadata` 中的 mtimeMs 返回值（可选保留）。

#### ⑥ `packages/ohbaby-agent/src/tools/edit.unit.test.ts` 和 `write.unit.test.ts`

重写测试：移除所有 mtimeMs 相关测试，新增：
- 连续编辑测试
- 并发编辑测试（多条编辑指令串行执行后文件内容正确）
- 模糊匹配测试（空格变化仍能匹配）
- 外部修改后编辑被拒绝测试
