# tokenCounting 模块的目标与职责

## 设计目标（Design Goals）

1. **轻量级的同步 Token 估算**
   - 提供无需外部依赖的 Token 计数能力
   - 支持同步执行，适合实时应用场景

2. **为 LLM 对话提供上下文感知**
   - 使系统能够了解当前会话的 Token 消耗状况
   - 帮助模块间的协作基于准确的资源认知

3. **主动警示机制**
   - 在达到 Token 限制时提前预警
   - 为对话管理提供清晰的决策依据

## 职责（Duties）

1. **Token 数量估算**
   - 根据文本内容（ASCII / 非 ASCII 字符）进行启发式 Token 估算
   - 对聊天消息及消息历史进行 Token 计算

2. **模型 Token 限额管理**
   - 维护不同 LLM 模型的 Token 限额映射表
   - 为新模型提供保守的默认限额

3. **会话 Token 使用情况计算**
   - 计算当前消息历史的 Token 消耗
   - 评估剩余可用 Token 容量

4. **Token 限制警告**
   - 检测是否接近或超过 Token 限制
   - 以清晰的严重程度（无告警 / 警告 / 严急）分类

## 非职责（Non-Duties）

1. **不执行精确 Token 计数**
   - 仅提供基于启发式算法的估算值
   - 精确的 Token 计数由 LLM API 响应提供

2. **不管理对话历史**
   - 不负责消息的截断、删除或缓存
   - 不参与对话流程的决策

3. **不持久化统计数据**
   - 不进行 Token 使用的数据存储
   - 不提供历史分析或成本统计功能

4. **不调用外部 API**
   - 所有计算基于本地启发式算法
   - 不依赖第三方 Token 计数服务

5. **不支持动态模型配置**
   - Token 限额由固定映射表定义
   - 不支持运行时修改模型的 Token 限制
