# 07 · v0.1.5 stable server kernel

> 本文把 `hono-app/06` 的 M1-M4 收束为 v0.1.5 发布前主任务。目标不是交付完整 web/app surface，而是先把显式 server 的内核稳定下来：结构可维护、事件投递可恢复、jsonrpc 兼容路径可继续服务 CLI remote。

---

## 1. Release Scope

v0.1.5 采用 **server 基础设施稳定版** 范围，只包含 `hono-app/06` 的 M1 through M4：

| Step | 交付 | 发布价值 |
|------|------|----------|
| M1 | 抽 `coordination/client-view.ts` | 把 per-client 视图从手写 HTTP server 中拿出来，避免后续 jsonrpc/web adapter 漂移 |
| M2 | 新建 Hono app 骨架 + 可注入 `app.fetch` | 让 server 有可测试的 app 装配点，但暂不切业务流量 |
| M3 | jsonrpc `/api/*` 迁到 Hono | 保持 `ohbaby --remote-port` 兼容，同时删除手写 route if-chain |
| M4 | 新增 event-bus + SSE replay | 修复断线静默丢事件问题，给 `/api/events` 带上 `id` / `Last-Event-ID` replay |

v0.1.5 的一句话目标：

> 默认 CLI 仍直连 backend；显式 server 改成 Hono 承载的兼容 jsonrpc server，并具备可恢复的 SSE 事件投递。

---

## 2. Explicit Non-Scope

以下继续 defer 到后续 web/app 接入版，不进入 v0.1.5：

| Deferred | 原因 |
|------|------|
| M5 web `/v1/*` REST routes | 需要 schema、CORS、typed client 与前端消费一起定，不混入 server 内核稳定版 |
| OpenAPI `/doc` 与 `@hono/zod-openapi` | 只有 web surface 启用时才有真实收益 |
| M6 InstanceStore / 全局单锁 / `x-ohbaby-directory` | 多项目 runtime 是下一阶段产品形态变化，本期不扩大生命周期面 |
| `serve ps` | 依赖 M6 的连接观测模型 |
| S8 fresh-lane 优化 | 与 M6 scope/lane key 设计绑定，本期只保留现状 |
| S9 disconnect 清待决 prompt | 与 prompt queue ownership/lane key 重构绑定；若 M4 发现必须修，只做最小兼容修复并单独记录 |

---

## 3. Invariants

- 默认 `ohbaby` / `ohbaby run` 继续走 in-process backend，不 import `hono`，不经过 `ohbaby-server`。
- `ohbaby serve` 与 `ohbaby --remote-port` 的用户可见行为保持兼容。
- Hono 依赖只进入 `packages/ohbaby-server`，不进入 `ohbaby-agent`。
- Auth 在 server 路由层 fail-closed：server 启动必须持有 token；请求缺失或不匹配时拒绝。
- SSE replay 不能静默丢事件：窗口内补发，窗口外发送 `resync-required` 控制事件。
- `/api/rpc`, `/api/events`, `/api/health`, `/api/shutdown` 保留为兼容 API。

## 4. Commit Batches

建议按以下批次提交：

1. `docs(server): scope v0.1.5 stable kernel`
2. `refactor(server): extract client view coordination`
3. `feat(server): introduce hono jsonrpc transport`
4. `feat(server): add replayable event bus`
5. `test(server): harden stable kernel verification`（如需要补充跨文件测试或回归门）

每个代码批次都要先写失败测试，再实现，再跑对应局部验证；最终再跑完整发布前验证。

---

## 5. Acceptance Gates

局部门：

- M1: `client-view` 单测覆盖 snapshot 投影、session event 过滤、command ownership、runtime ownership。
- M2/M3: Hono app health、auth fail-closed、jsonrpc RPC、shutdown、旧 `/api/events` 兼容 SSE 均有测试。
- M4: event-bus 单测覆盖 seqNum、窗口内 replay、窗口外 `resync-required`、实时订阅；集成测试覆盖 `Last-Event-ID` 重连。

发布前门：

```bash
pnpm --filter ohbaby-server build
pnpm run typecheck
pnpm run test:unit
pnpm run test:integration
pnpm run build
rg "from \"hono\"|from 'hono'" packages/ohbaby-cli packages/ohbaby-agent
```

最后一条 `rg` 预期无输出，用来证明默认 CLI 和 agent 没有引入 Hono。
